# Plot Standardization — Current Usage and API

## Overview

This page documents the current, stable plot standardization API used by the
simulation benchmarks and kidney case studies. The goal is to describe how to
produce the canonical figure set and how to reproduce plots from run manifests.

## Concepts

- Plot thresholds: canonical cutoffs are `C_cut` (coverage) and `S_cut`
  (spatial score). Defaults are `C_cut=0.30` and `S_cut=0.15` but these may be
  overridden by train-derived thresholds stored alongside run outputs.
- PlotSpec: a small serializable structure (see `biorsp.plotting.spec.PlotSpec`)
  containing cutoff and color information. Manifests may include a `plot_spec`
  entry to enable deterministic replotting.
- Standard plot set: generated by `biorsp.plotting.standard.make_standard_plot_set`
  which returns a mapping of figure identifiers to saved paths.

## How to reproduce canonical plots

1. If you have a run manifest containing `plot_spec`, use
   `biorsp.plotting.replot` to recreate figures without re-scoring data:

```bash
python -m biorsp.plotting.replot --manifest outputs/archetypes/manifest.json
```

2. To generate the canonical set programmatically from a `scores_df` DataFrame,
   call `make_standard_plot_set(scores_df, outdir, thresholds=...)`. The
   function is resilient to missing optional inputs (truth labels, embedding,
   etc.) and writes a small set of PNGs to `outdir`.

## API reference (short)

- `make_standard_plot_set(scores_df, outdir, thresholds=None, truth_col=None, pred_col=None, ...)`
  - Returns dict[str, Path] mapping figure ids (e.g. `fig_cs_scatter`) to files.
- `plot_cs_scatter(df, ...)` — Coverage vs Spatial Score scatter used in the
  canonical set.
- `plot_confusion_matrix(cm, labels, normalize='true')` — Small helper for
  confusion/composition visualizations.

## Notes

- Use `PlotSpec` from `biorsp.plotting.spec` when recording plot-related
  parameters in manifests to ensure reproducible color and cutoff choices.
- The standard plotting tools avoid raising on missing optional inputs; they
  will instead emit placeholder panels that indicate missing data.

## Examples

See the `biorsp.plotting` package for examples and the `replot` CLI for
reproducing figures from manifests.
